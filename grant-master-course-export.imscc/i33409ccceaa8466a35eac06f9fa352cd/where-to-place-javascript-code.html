In order for JavaScript to execute in the browser it must be found in one of three locations.

Between <script> tags within the HTML document.

You can place any number of scripts in an HTML document. Scripts can be placed in the <body>, or in the <head> section of an HTML page, or in both. Placing scripts at the bottom of the <body> element improves the display speed, because script compilation slows down the display.

As a separate file.

The script tags do not need to be included in the file. However a link to the file must be included in the web page. 

myScript.js

function myFunction() {
 document.getElementById("demo").innerHTML = "Paragraph changed.";
}

You can place an external script reference in <head> or <body> as you like. The script will behave as if it was located exactly where the <script> tag is located.

<script src="myScript.js"></script>

To add multiple script files to one page - use multiple script tags:

<script src="myScript.js"></script>
<script src="myOtherScript.js"></script>

 

As an External Resource

External scripts are those scripts that are hosted on another computer or server. They can be referenced with a full URL or with a path relative to the current web page. The script tag should be placed within the head tags of the current web page.

<script src="https://www.domain.com/js/myScript.js"></script>

External JavaScript Advantages

  • It separates HTML and code
  • It makes HTML and JavaScript easier to read and maintain
  • Cached JavaScript files can speed up page loads
